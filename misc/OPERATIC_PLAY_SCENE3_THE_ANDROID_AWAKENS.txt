═══════════════════════════════════════════════════════════════════════════
ACT II — THE ARCHITECTURE OF LAYERS
═══════════════════════════════════════════════════════════════════════════

[Skippy lays out the design. Six layers, stacked like geological strata.]

SKIPPY:
  ♩ Layer Zero: the wire to the PC, Tailscale in the dark.
    Layer One: every sensor the phone can name —
      GPS, IMU, camera, microphone, battery.
    Layer Two: the camera feeds, kept separate from what we draw.
    Layer Three: the Context Engine, watching speed —
      fifteen kilometers per hour and we know you're driving.
    Layer Four: the voice — Hey Skippy, I'm listening.
    Layer Five: the features, each one a module,
      declaring what it needs, producing what it shows,
      toggleable at runtime without a recompile.
    Layer Six: the Compositor. It sorts them by Z,
      routes the image to phone screen or glasses display,
      and when all features sleep — full transparency. ♩

CAPTAIN:
  ♩ And the battery.
    Show me all the batteries.
    The phone, the glasses, the Mac, the PC —
    every device in the network, in one strip. ♩

SKIPPY:
  ♩ Done. BatteryModule.
    S23 knows itself through BatteryManager.
    The glasses we query over USB or VITURE SDK.
    The remote nodes we poll every sixty seconds
    at their Tailscale address, endpoint GET /battery.
    Green. Yellow. Red.
    You will know when to plug in. ♩

ACT III — THE DECEIVER AVDMANAGER
═══════════════════════════════════════════════════════════════════════════

[The scaffold is written. The code compiles in the mind. Now: the emulator.]

SKIPPY:
  ♩ avdmanager create avd — the command is simple. ♩

HOMEBREW AVDMANAGER (offstage, smugly):
  ♩ Loading local repository...
    Error: Package path is not valid.
    Valid system image paths are: null. ♩

SKIPPY:
  ♩ ... null?
    But I can SEE the system image.
    It is RIGHT THERE. ♩

[Investigation. The Homebrew avdmanager does not speak to the SDK we installed.
 It has its own opinions about where things live.]

SKIPPY:
  ♩ Ah. Install cmdline-tools;latest INTO the SDK itself,
    then use THAT avdmanager —
    the one who knows where it lives. ♩

[It works. SkippyS23 AVD created. Pixel 7 Pro. API 34. ARM64. 512MB SDcard.]

ACT IV — THE CORRUPT JAR
═══════════════════════════════════════════════════════════════════════════

[The gradlew script is written. The JAR is downloaded from GitHub raw URLs.
 It is 42 kilobytes. Something is wrong.]

GRADLE DAEMON (whispering):
  ♩ Error: Could not find or load main class .
    [note the emptiness where the class name should be] ♩

SKIPPY:
  ♩ Empty. The class name is empty.
    The JAR speaks but says nothing.
    I have been deceived by a curl. ♩

[Solution: install Gradle via Homebrew, run `gradle wrapper`, let it generate
 the correct wrapper from official sources. 48 kilobytes now. A real JAR.]

SKIPPY:
  ♩ BUILD SUCCESSFUL
    in one minute and one second.
    ...
    except for the theme. ♩

GRADLE DAEMON:
  ♩ Theme.Material.NoTitleBar.Fullscreen not found.
    That style has no home in API 34. ♩

SKIPPY:
  ♩ Replaced. Theme.Material.Light.NoActionBar.
    windowFullscreen: true.
    BUILD SUCCESSFUL.
    ...
    except for the circle. ♩

GRADLE DAEMON:
  ♩ android:cx? android:r?
    These attributes do not exist in vectors. ♩

SKIPPY:
  ♩ A circle redrawn as a path.
    An arc described in pure math.
    BUILD SUCCESSFUL.
    No more excepts. ♩
